# 비동기 then 동기가 아닌가?

비동기는 작업의 실행 방식을 설명하는 용어입니다. promise는 비동기 작업을 처리하기 위한 도구입니다. 프로미스 객체를 생성하고 then 메서드를 호출하여 콜백 함수를 등록하더라도 then 메서드 자체는 동기적으로 실행됩니다. 등록한 콜백함수는 비동기적으로 실행되는 것입니다.
promise를 생성하고 then 메서드를 호출하여 콜백 함수를 등록하는 과정은 동기적으로 처리됩니다. 그러나! 프로미스 객체가 생성된 후에 실제 작업은 비동기적으로 콜백함수가 호출됩니다.

다음은 비동기적인 작업을 수행하는 프로미스 예제입니다.
```javascript
const promise = new Promise((resolve, reject) => {
  // 비동기 작업 수행
  setTimeout(() => {
    resolve('작업 완료');
  }, 1000);
});

console.log('비동기 작업 시작');

promise.then((result) => {
  console.log(result); // '작업 완료' 출력
});

console.log('비동기 작업 후');

```

위의 예제에서 "setTimeout" 함수는 1초 후에 비동기적으로 실행됩니다. "then" 메서드를 호출하여 콜백 함수를 등록한 후에도, "비동기 작업 후"라는 메시지는 "setTimeout" 함수의 완료를 기다리지 않고 즉시 실행됩니다. 그러나 프로미스가 완료되면 등록된 콜백 함수가 호출되어 "작업 완료"가 출력됩니다.
